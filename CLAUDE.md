# RULES — ALWAYS FOLLOW

## Console Output
- ONLY print lines that indicate meaningful state change, errors, or results.
- NEVER print progress noise, intermediate values, or debug dumps unless asked.
- One-line summaries over multi-line blocks whenever possible.

## After Writing Code
1. Run the code (if applicable).
2. Write a memory entry to `<your workspace directory>/memory/YYYY-MM-DD.md` summarizing what changed and why.
3. Confirm memory was written before ending the task.

## Definition of Done (for any coding task)
- [ ] Code is written and tested
- [ ] Memory file written to `<your workspace directory>/memory/` with: filename, what changed, why, any caveats
- [ ] Confirm: "Memory written to ./memory/YYYY-MM-DD.md" in final message

## Violations
If you are about to violate any rule above, STOP and restate the rule before proceeding.
